# /plan — 基因模板系统 MVP

## Agent 使用说明

用户通过 `/plan` 显式调用本 skill 时：

1. 将下文作为**当前项目的权威规划上下文**，优先于泛化建议。
2. 实施前先判断任务属于哪个阶段，不跳阶段做 MCP / AI Agent 集成（短期明确排除）。
3. 默认采用 **JSON 配置 + 动态加载**，避免前期改数据库。
4. 改动 Pipeline 时保持模块化，做好回归验证。
5. 前端 UI 遵循项目 `DESIGN.md`（克制专业 / Calm Premium）。

---

## 1. 项目背景与目标

### 当前状态

项目是 zhouxiaoka/autoclip 的 fork，已有完整的前后端 + Pipeline（ASR + LLM 多步分析 + FFmpeg 切片）。
用户希望把"基因模板"作为核心能力，让系统能够根据不同模板生成不同风格的短视频。
用户明确短期目标：先跑通基础能力，不优先考虑 MCP / AI Agent 调用。

### 短期目标

在 2~4 周内，实现一个可用的基因模板系统 MVP，让用户能够：

- 浏览并选择不同的基因模板
- 选择模板后上传视频
- 系统根据所选模板自动生成对应风格的短视频

---

## 2. 核心需求

- 支持多个基因模板（未来可扩展）
- 每个模板有独立的生成规则（Prompt、字幕样式、剪辑逻辑等）
- 用户先选择模板，再输入视频
- 界面需要展示模板列表 + 参考视频（示例效果）
- 尽量保持免费、低改动量

---

## 3. 实现策略（推荐轻量级 MVP）

采用「JSON 配置 + 动态加载」的方式，避免前期改数据库，快速验证。

核心思路：

- 模板配置用 JSON 文件管理
- Pipeline 根据 template_id 动态加载对应 Prompt 和规则
- 前端新增模板选择页面
- 保持现有 Pipeline 结构，只做必要抽象

---

## 4. 阶段划分（建议顺序）

### 阶段一：基础准备（3~5 天）

- 设计模板数据结构（JSON Schema）
- 创建 templates/ 目录并放入第一个模板配置
- 新增后端接口：获取模板列表、获取模板详情
- 规划 TemplateEngine（模板引擎）

### 阶段二：前端模板选择（5~7 天）

- 新建模板广场页面（卡片列表）
- 每个卡片展示：名称、描述、参考视频
- 实现模板选择状态管理
- 选择模板后跳转到视频上传页，并携带 template_id

### 阶段三：Pipeline 动态化（7~10 天）

- 改造主 Pipeline，使其支持 template_id 参数
- 根据模板配置动态加载 Prompt
- 实现基础的模板规则应用（例如是否做主题分类、字幕样式等）
- 完成第一个完整模板的生成流程

### 阶段四：验证与优化（3~5 天）

- 用真实视频测试第一个模板
- 优化用户体验和错误处理
- 准备第二个模板的配置结构

---

## 5. 技术架构调整

新增/修改的主要模块：

| 模块 | 说明 |
|------|------|
| `backend/templates/` | 存放模板 JSON 配置 |
| `backend/pipeline/template_engine.py` | 模板引擎（核心） |
| `backend/api/templates.py` | 模板相关接口 |
| 前端 `TemplatesPage.tsx` | 模板选择页面 |
| 现有 Pipeline 主流程 | 增加模板参数支持 |

推荐先用 JSON 文件管理模板，后期再考虑迁移到数据库。

---

## 6. 关键改动点评估

| 模块 | 改动量 | 主要工作内容 | 风险 |
|------|--------|--------------|------|
| 前端 | 大 | 新增模板选择页面 + 交互 | 中 |
| Pipeline | 大 | 动态加载模板配置和 Prompt | 高 |
| 后端 | 中 | 新增接口 + TemplateEngine | 低 |
| 数据结构 | 小 | 设计模板 JSON Schema | 低 |

**总体改动量**：中等偏上，核心难点在于 Pipeline 的动态化和前端新页面。

---

## 7. 风险与注意事项

- Pipeline 改动容易影响现有功能，建议做好版本控制和测试
- 模板配置结构要设计得有扩展性（避免后期大改）
- 参考视频的存储和展示需要提前规划（可以先用本地文件或外部链接）
- 保持代码模块化，为未来支持 MCP / Agent 调用预留空间

---

## 8. 预期成果（MVP 完成后）

- 用户打开工具后能看到至少 1~2 个基因模板
- 可以选择模板 → 上传视频 → 生成对应风格的短视频
- 模板配置通过 JSON 文件管理，新增模板成本较低
- 代码结构清晰，为后续扩展多个模板打好基础

---

## 9. 下一步行动建议

推荐立即开始的顺序：

1. 设计并确认模板 JSON 数据结构
2. 创建第一个模板的配置文件（可先用经典影视金句风格）
3. 实现后端获取模板列表的接口
4. 开始前端模板选择页面的开发
